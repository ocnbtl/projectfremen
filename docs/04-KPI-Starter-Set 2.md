# KPI Starter Set (Pre-Revenue Friendly)

These KPIs are designed for early-stage brands that are not fully launched yet.

## Dashboard Priorities

Use a simple three-level priority:

1. P1 (must track weekly)
2. P2 (track weekly or biweekly)
3. P3 (track monthly)

## Cross-Brand Core KPIs

1. P1: Weekly execution completion rate (% tasks completed on time)
2. P1: Open blockers count
3. P1: Weekly shipping activity (new meaningful commits or releases)
4. P2: Active workstreams count
5. P2: Stale critical docs count (no update beyond review cadence)
6. P3: Monthly strategic progress score (self-rated)

## pngwn (Project Iceflake) Starter KPIs

1. P1: Website waitlist signups per week
2. P1: Website conversion rate (visitor to signup)
3. P2: Feature readiness progress (% of MVP checklist done)
4. P2: Brand asset readiness score
5. P3: Legal/admin readiness score

## Ranosa Decor (Project Pint) Starter KPIs

1. P1: Content output shipped (pins/posts per week)
2. P1: Outbound click performance trend
3. P2: Approved content backlog size
4. P2: Operating packet completion status
5. P3: Monetization readiness score

## Unigentamos (Project Fremen) Starter KPIs

1. P1: Dashboard implementation velocity (milestones hit / week)
2. P1: Documentation coverage (% of 12 business areas with current notes)
3. P2: Compliance calendar completeness
4. P2: Decision log freshness (days since last key decision logged)
5. P3: System reliability score (manual)
